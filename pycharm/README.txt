Программа запускається через PyCharm 2025.1.1.1 з правами адміністратора.

1. Розпакувати архів:
Перш за все, потрібно розпакувати архів (наприклад, .zip або .rar) у будь-яку зручну папку на його комп'ютері.

2. Встановити Python (якщо ще не встановлено):
Переконатися, що на його комп'ютері встановлений Python(наприклад, Python 3.9, 3.10, 3.11, 3.12). Якщо Python не встановлено, його потрібно завантажити з офіційного сайту python.org та встановити.
Важливо: Під час встановлення Python для Windows рекомендується поставити галочку "Add Python X.Y to PATH". Це спростить запуск скриптів з командного рядка.

3. Встановити Npcap (для Windows, якщо ще не встановлено):
Бібліотека Scapy, яку використовує твій проект для захоплення пакетів, потребує Npcap на Windows.
Завантажити Npcap з офіційного сайту: https://npcap.com/#download (зазвичай потрібен останній стабільний реліз).
Під час встановлення Npcap важливо обрати опції:
* "Support raw 802.11 traffic (and monitor mode) for wireless adapters" (опціонально, але корисно, якщо планується робота з Wi-Fi у режимі моніторингу).
* "Install Npcap in WinPcap API-compatible Mode" (важливо для сумісності зі Scapy, хоча новіші версії Scapy можуть працювати і без цього, але це безпечний варіант).
* "Restrict Npcap driver's access to Administrators only" (рекомендовано з міркувань безпеки, але тоді програму завжди треба буде запускати від імені адміністратора).
* "Support loopback traffic ("Npcap Loopback Adapter")" (може бути корисним для тестування).

4. Створити та активувати віртуальне середовище (рекомендовано):
Це найкраща практика, щоб ізолювати залежності проекту від глобального Python.
Відкрити командний рядок (cmd або PowerShell) у папці, куди було розпаковано проект.

*   Для PowerShell:
    ```bash
    .venv\Scripts\Activate.ps1
    ```
Після активації, на початку командного рядка з'явиться (.venv).
5. Встановити необхідні бібліотеки Python:
У тебе в проекті, ймовірно, є файл requirements.txt, який містить список усіх залежностей. Якщо так, то в активованому віртуальному середовищі потрібно виконати:
bash pip install -r requirements.txt
Якщо файлу requirements.txt немає, то друг повинен буде встановити бібліотеки вручну. З нашого обговорення, це щонайменше:
bash pip install customtkinter scapy requests

8. Запустити програму від імені адміністратора:
Це критично важливо для коректної роботи Scapy (захоплення пакетів) та firewall_manager.py (додавання правил у Брандмауер Windows).
Як запустити з командного рядка від імені адміністратора:
1. Знайти "Командний рядок" (Command Prompt) або "PowerShell" у меню "Пуск".
2. Клікнути правою кнопкою миші -> "Запуск від імені адміністратора".
3. У вікні, що відкрилося, перейти до папки з проектом за допомогою команди cd шлях_до_папки_проекту.
4. Активувати віртуальне середовище (як описано в пункті 4).
5. Запустити головний скрипт:
python main_app.py
